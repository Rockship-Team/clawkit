# Outline Quality Guide

## Structure Standards

### Required Sections (in order)
1. **Title Page** - Company info, proposal title, date, validity
2. **Executive Summary** - 2-3 sentences: objective + impact
3. **Understanding Your Needs** - Current state, challenge, goal, scale
4. **Proposed Solution** - Use cases, capabilities, deployment, integrations
5. **Investment Options** - 3 tiers: Starter/Pro/Enterprise
6. **Implementation Timeline** - Phases, duration, deliverables
7. **Expected Results** - Metrics table with before/after comparison
8. **Why {Vendor Name}** - 3 differentiators
9. **Next Steps** - Clear action items

### Optional Sections (add if relevant)
- **Risk Assessment** - Risks + mitigations table
- **Team Overview** - Project team bios
- **Case Studies** - Similar successful projects
- **Technical Specifications** - Detailed tech stack
- **Support & Maintenance** - Ongoing service details
- **Legal & Compliance** - Terms, SLAs, guarantees

## Quality Criteria

### Completeness
- [ ] Each section has substantive content (not filler)
- [ ] All client requirements addressed
- [ ] Pricing options complete with features
- [ ] Timeline realistic and detailed
- [ ] Metrics quantified with numbers

### Consistency
- [ ] Terminology consistent throughout
- [ ] Currency format uniform (VND/year or monthly)
- [ ] Date formats consistent (DD/MM/YYYY)
- [ ] Number formatting consistent (1,000,000 vs 1M)
- [ ] Headings hierarchy clear (H1 > H2 > H3)

### Clarity
- [ ] Plain language (no jargon without explanation)
- [ ] Active voice preferred ("we will build" vs "will be built")
- [ ] Short paragraphs (max 4-5 lines)
- [ ] Tables where comparisons needed
- [ ] Bullet points for lists
- [ ] Bold key numbers/metrics

### Accuracy
- [ ] Company name spelled correctly (check COSMO)
- [ ] Contact info verified
- [ ] Pricing matches pricing-packages.md exactly
- [ ] Timeline realistic for scope
- [ ] Metrics grounded in benchmarks (don't invent)

## Section Guidelines

### Executive Summary
- **Length:** 2-3 sentences max
- **Content:** Business objective + expected impact + key metric
- **Tone:** Confident, value-focused
- **Example:** "Rockship proposes deploying an AI-powered customer service platform for ABC Corp to reduce response time from 2 hours to 30 seconds, automating 70% of tickets and saving 150 monthly support hours."

### Understanding Your Needs
- **Length:** 1 paragraph + 4-6 bullet points
- **Content:** Where they are now, main challenge, desired outcome, scale metric
- **Tone:** Empathetic, diagnostic
- **Example:** 
  ```
  ABC Corp's customer support team is overwhelmed by 300+ daily inquiries, 
  resulting in 2-hour response times and declining CSAT scores. Their 
  goal is to maintain service quality while reducing operational costs. 
  Current scale: 300 inquiries/day across 3 channels (email, chat, phone)
  ```

### Proposed Solution
- **Length:** 4-6 use cases + 4-6 capabilities + deployment table
- **Content:** Specific features mapped to pain points
- **Tone:** Solution-oriented, confident
- **Structure:** Use cases (what), capabilities (how), deployment (where)

### Investment Options
- **Length:** 3 packages with 5-7 features each
- **Content:** Price, features included, why this fits, trade-offs
- **Tone:** Value-focused, not cost-focused
- **Examples:**
  - **Starter:** "Perfect for testing and validation"
  - **Pro:** "Recommended for established businesses"
  - **Enterprise:** "For organizations requiring custom solutions"

### Implementation Timeline
- **Length:** 4-5 phases, 1-2 sentences each
- **Content:** What happens in each phase, key deliverables
- **Tone:** Realistic, confidence-building
- **Structure:** Discovery → Development → Testing → Launch → Handover
- **Duration:** Must match package complexity (1-4 weeks for AI agents, 3-6 months for custom dev)

### Expected Results
- **Length:** Table with 4-6 metrics
- **Content:** Current state → projected state → improvement %
- **Tone:** Conservative but optimistic (use benchmarks)
- **Examples:**
  - Response time: 2 hours → 30 minutes (75% improvement)
  - Automation rate: 0% → 65%
  - CSAT score: 3.5 → 4.5 (28% improvement)
  - Monthly support costs: - → -30% reduction

### Why Vendor
- **Length:** Exactly 3 bullet points
- **Content:** 1 language/localization, 1 speed/deployment, 1 support/tracking
- **Tone:** Confident but not arrogant
- **Examples:**
  - "Native Vietnamese NLP with 95% accuracy"
  - "Go-live in 3 weeks, not 3 months"
  - "Dedicated success manager with 24/7 support"

### Next Steps
- **Length:** Exactly 4 action items
- **Content:** Clear, numbered, with emojis/icons
- **Tone:** Urgent but not pushy
- **Examples:**
  1. ✅ Review and approve proposal
  2. 📞 Schedule technical deep-dive (30 min)
  3. 💰 Select investment option
  4. 🚀 Kick-off Discovery phase (Week 1)

## Language Guidelines

### Vietnamese Content
- **Tone:** Professional yet friendly (không quá trang trọng, không quá suồng sã)
- **Currency:** VND with "triệu" abbreviation (20 triệu VND, 150M VND)
- **Dates:** DD/MM/YYYY format
- **Measurement:** Uses metric system (kg, km, giờ, phút)
- **Examples:**
  - "ROI +233% trong 3 tháng" (not "ROI tăng 233%")
  - "Go-live trong 4 tuần" (not "Phát hành trong 1 tháng")
  - "Cam kết SLA 99.9%" (not "Đảm bảo uptime 99.9%")

### English Content
- **Tone:** Professional, direct
- **Currency:** $X,XXX for thousands, $X.X million for millions
- **Dates:** MM/DD/YYYY
- **Measurement:** Mix (hours for time, kg/lbs depending on audience)
- **Examples:**
  - "ROI +233% in 3 months"
  - "Go-live in 4 weeks"
  - "SLA 99.9% guaranteed"

## Common Mistakes to Avoid

### ❌ Content Mistakes
- Inventing client information not from COSMO/user input
- Exposing walk-away pricing to clients
- Making unrealistic claims (e.g., "100% ROI")
- Using competitor names negatively
- Overloading with technical jargon

### ❌ Formatting Mistakes
- Inconsistent table styles
- Mixed currency formats
- Missing validity date
- No table of contents (for long proposals)
- Poor spacing (too cramped or too sparse)

### ❌ Strategic Mistakes
- Leading with price (lead with value)
- Not addressing client's stated pain points
- Ignoring missing info flagging
- One-size-fits-all language (customize per company)
- Forgetting ROI calculations

## Review Checklist

Before showing to client:
- [ ] Company name and contact info correct
- [ ] All client requirements addressed
- [ ] Pricing matches pricing-packages.md
- [ ] Timeline realistic and clear
- [ ] ROI calculations sound and conservative
- [ ] No internal notes accidentally included
- [ ] Vietnamese/English language consistent throughout
- [ ] Formatting clean and professional
- [ ] Missing info clearly flagged
- [ ] Next steps clear and actionable

3. Test with sample data
4. Version control changes
